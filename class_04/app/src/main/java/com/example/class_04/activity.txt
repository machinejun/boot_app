Activity
1. 앱의 한 화면입니다.
2. 첫 화면에서 다른 화면으로 넘어 간다는 것은 Activity 가 바뀌는 것이다.
3. Life Cycle(수명 주기) : 생성 & 소멸이 있다.
4. 콜백 : 특정한 동작이 발생 되면 알려 준다.


OnCreate
: activity 가 만들어 질 때 단 한번만 호출 된다.
: 단 한번만 해야 되는 작업 여기서 해주면 된다.

OnStart
: 호출되면 화면이 사용자에게 표시 되고 화면에 포그라운드에 보내져서
  사용자와 상호작용을 할 수 있도록 준비한다.
  = 셋팅 작업

OnResume
: 다시 앱으로 돌아 올 때 무조건 호출 된다.
: activity 가 다시 호출 될 때 하면 되는 작업들을 여기서 해주면 된다.
ex> 카카오톡을 다시 켰을 때 상대방이 보냈었던 메세지를 보여준다.

onPause
: 화면의 일부가 가려 졌을 때

onStop
: 화면의 전부가 보이지 않을 때
: 임시 데이터들을 저장해 놓을 수 있다.

onDestroy
: 액티비티가 완전히 소멸 되기 전에 호출 된다.
: 사용자가 화면을 닫거나 종료 기능을 눌렀을 때 호출
-> 구성 변경 ( 기기 화면 or 멀티 윈도우 모드)로 인해 시스템에서 일시적으로
   액티비티를 소멸 시키는 경우
   >> 화면을 돌렸을 때 현재 프로그램을 정지시키고 다시 프로그램 재개하는 방식으로
      레이아웃을 그려준다.
